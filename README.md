# Advanced RAG with LangChain

An advanced Retrieval-Augmented Generation (RAG) pipeline built with LangChain, ChromaDB, and locally-hosted LLMs via Ollama. The notebook ([main.ipynb](main.ipynb)) demonstrates how naive RAG falls short and layers in several techniques to improve retrieval quality and answer precision.

## What It Does

The pipeline answers questions about the **2024 State of the Union Address** by loading the Wikipedia article and putting it through a multi-stage RAG flow.

### Pipeline Overview

```
User Query
    │
    ▼
1. Pre-Retrieval: Query Rewriting
    │   Rewrites the raw query into a richer semantic statement
    │   to improve vector similarity search accuracy
    ▼
2. Retrieval
    │   Semantic search over ChromaDB vector store
    │   (embeddings: qwen3-embedding:8b via Ollama)
    ▼
3. Post-Retrieval: Rerank + Extract (Compression Pipeline)
    │   ├── FlashrankRerank  — reorders chunks, best matches first
    │   └── LLMChainExtractor — strips irrelevant sentences from each chunk
    ▼
4. Generation
    │   Final prompt assembled from compressed context + original query
    └── Answer generated by gemma4:e4b (reasoning mode, via Ollama)
```

### Why Advanced?

| Naive RAG | Advanced RAG (this repo) |
|-----------|--------------------------|
| Raw query sent directly to vector store | Query is rewritten to improve semantic match |
| All retrieved chunks passed to LLM | Chunks reranked by relevance score (Flashrank) |
| Full chunk text in context | LLM extractor removes irrelevant sentences per chunk |

## Stack

| Component | Tool |
|-----------|------|
| Document loading | `langchain_community.document_loaders.WebBaseLoader` |
| Text splitting | `RecursiveCharacterTextSplitter` (1000 chars, 200 overlap) |
| Embeddings | `qwen3-embedding:8b` via Ollama |
| Vector store | ChromaDB (HTTP server on `localhost:8000`) |
| Reranker | `FlashrankRerank` (langchain-classic) |
| LLM extractor | `LLMChainExtractor` (langchain-classic) |
| LLM | `gemma4:e4b` via Ollama (reasoning mode) |
| Orchestration | Custom `RAGChain` class |

## Requirements

- [Ollama](https://ollama.com/) running locally with `gemma4:e4b` and `qwen3-embedding:8b` pulled
- ChromaDB HTTP server running on `localhost:8000`
- Python dependencies (install via pip):

```bash
pip install langchain langchain-community langchain-chroma langchain-ollama langchain-classic chromadb flashrank
```

## Running

Open [main.ipynb](main.ipynb) in Jupyter and run cells top to bottom. The notebook:

1. Loads and chunks the Wikipedia article
2. Embeds chunks and stores them in ChromaDB
3. Validates retrieval with a raw similarity search
4. Builds the compression retriever (rerank + extract)
5. Runs two example queries through the full advanced RAG chain
